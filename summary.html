User Research

User Interviews

User interviews are an important part of the user research process because they help you target the specific features hiring managers or potential viewers are interested in seeing. The types of questions asked during a user interview should be professional if talking to a hiring manager but informal enough to get their personal opinions about what THEY are looking for in an online porfolio.

Competitor Analysis

A competitor analysis is looking at other portfolios or websites that might have similar intentions as your own. By conducting competitor analyses, you can get a clearer picture about the content that your competitors have on their websites. This is a great way to draw inspiration and ideas for what to include in your own portfolio/website.

Feature Value Matrix

A feature value matrix is a technique where you incorporate and map values/numbers to certain types of features you want to implement in your website/portfolio. This helps clarify which features are worth implementing and are the most crucial/important to include. For my feature matrix, I chose to include the features that were mentioned to me in the User Interviews I conducted. These two User Research methods/tactics go hand in hand.

Feature Priortization

Through the feature value matrix, prioritizing features was as simple as seeing which values came out to be more important! When building an interface, it is important to know which features are the most important and valuable to implement because writing code is a costly, time consuming effort. Being efficient and resourceful can not only save time and money, but it can also make the interface less bombarded with information that might be redundant or useless.

User Testing

User Stories

User Stories are a great way to verify if your features are implemented exactly the way you want them to be. Often times, implemented features end up behaving in ways different from what the developer intended. User Stories allow the developer to put themselves in the shoes of their possible users and see if their features would behave as expected.
